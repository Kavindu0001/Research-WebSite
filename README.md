## 🚀 Getting Started

### Local Development
1. Download all files and maintain the folder structure
2. Open `index.html` in a web browser to view the website
3. Navigate using the menu at the top of each page

### File Paths
- All relative paths are set up correctly
- CSS files are linked from the `assets/css/` directory
- Pages are organized in the `pages/` subdirectory

## 📝 Customization Guide

### Adding Content
1. **Replace dummy content** with actual project information
2. **Update team member information** in `about-us.html`
3. **Add document links** in `documents.html` (modify `#` href to actual file paths)
4. **Update contact details** in `contact-us.html`

### Modifying Colors
Edit the following variables in `assets/css/style.css`:
- Primary color: `#3498db` (blue)
- Dark color: `#2c3e50` (dark gray)
- Light color: `#ecf0f1` (light gray)

### Adding New Pages
1. Create a new HTML file in the `pages/` directory
2. Copy the header and footer structure from existing pages
3. Update navigation links to include the new page
4. Maintain consistent styling using the same CSS file

## 🌐 Hosting Preparation

### For WordPress Hosting
1. Convert HTML files to WordPress theme files
2. Create a custom post type for documents/slides
3. Set up a contact form plugin
4. Configure media library for team photos

### For Static Hosting (GitHub Pages, Netlify)
1. Upload all files maintaining the folder structure
2. Ensure index.html is in the root directory
3. Test all links work correctly
4. Configure custom domain (if applicable)

### For Traditional Web Hosting
1. FTP/SFTP upload all files
2. Ensure proper file permissions (644 for files, 755 for directories)
3. Update absolute URLs if domain changes
4. Set up SSL certificate for HTTPS
